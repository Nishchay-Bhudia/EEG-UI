Bluetooth Headband
—

Works with Muse, BrainBit, and any BLE EEG headband. Raw EEG is collected for 2 s then sent to your Render backend for full Random-Forest classification — or processed locally if no backend is configured. Requires Chrome or Edge on desktop or Android. Safari, Chrome, and every other browser on iOS/iPadOS can't do this at all — Apple doesn't implement Web Bluetooth. On iPhone/iPad, use a Web-Bluetooth-capable browser app instead, such as "Bluefy – Web BLE Browser" (search the App Store). In Bluefy the device picker shows every nearby Bluetooth device, not just headbands, because iOS won't pre-filter the list the way desktop/Android Chrome does — just pick the one named "Muse…" from the list. Sleep Mode keeps the screen on (dimmed near-black) instead of letting it lock — this is a hard limit of the web, not this app: every browser, including Bluefy, stops running entirely once the screen actually locks, so there's no way to record with the screen truly off.

Render Backend URL

The backend must expose POST /analyze (raw EEG) and optionally POST /analyze/bands (pre-computed powers).

Raw EEG endpoint
POST /analyze
{
  "eeg_data": [[ch0_s0, ch0_s1, ...], [ch1_s0, ...]],
  "sample_rate": 256
}
Pre-computed bands endpoint
